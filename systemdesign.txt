SHOPAI SYSTEM DESIGN DOCUMENT
=============================

Project summary
---------------
ShopAI is a three-tier e-commerce platform with:
- React frontend for customer and admin UI
- ASP.NET Core backend for authentication, catalog, cart, orders, address, admin, QR payment, and AI bridge endpoints
- Python FastAPI AI agent for conversational shopping, semantic product search, and voice transcription

The backend is the central business API. The frontend talks to the backend for normal commerce features and talks to the Python AI agent for chat and speech. The AI agent, in turn, calls the backend AI routes to read or modify cart, orders, addresses, and products.

-------------------------------

- (Terminator)  Start / End

- <Decision?>   Branching logic with Yes / No outcomes

- \Output\     Displayed result, response payload, notification, or generated artifact

4. Low Level Design Pseudocode
------------------------------

4.1 Authentication and Session Workflow
---------------------------------------
Pseudocode:

START
	read token, user, bootId from sessionStorage/localStorage
	if token and user exist:
			call backend validate endpoint with bearer token
			if validation fails:
					clear session and redirect to home
			else:
					compare stored bootId with current server bootId
					if mismatch:
							clear session and redirect to home
					else:
							rebuild user object and continue
	else:
			show public pages
END


4.2 Login and Registration
--------------------------
Pseudocode:

START
	user submits login or registration form
	validate required fields on client
	if login:
			send email and password to backend auth login endpoint
			if JWT and role returned:
					check role matches selected login type
					extract userId from JWT or response
					store token, user, and bootId in sessionStorage and localStorage
					route user by role
			else:
					show error
	if register:
			send fullName, email, mobile, password, gender to backend register endpoint
			if successful:
					store token, user, and bootId
					initialize profile data
					route user to products
			else:
					show error
END


4.3 Product Catalog and Search
------------------------------
Pseudocode:

START
	load all products from backend Mongo product endpoint
	normalize response fields so frontend always gets id, name, price, imageUrl, category, stockQuantity
	when search query changes:
			call backend semantic search endpoint with query and topK
			receive ranked product list
			map each result to a uniform product object
	render products in grid and allow opening product details
END


4.4 AI Chat Assistant
---------------------
Pseudocode:

START
	user enters chat text or voice input
	if voice input:
			send audio to speech transcription endpoint
			replace input text with transcript
	send message, userId, history, and JWT to AI agent /chat
	AI orchestrator loads semantic search service and agent tools
	LLM inspects user intent
	if action requires product lookup:
			call semantic search
	if action requires cart/order/address data:
			call backend AI routes through authenticated API client
	trim sensitive data from tool results
	generate final reply and optional product cards
	return reply to frontend
END


4.5 Shopping Cart
-----------------
Pseudocode:

START
	if authenticated user exists:
			fetch cart from backend by userId
			cache cart in sessionStorage
	else:
			load guest cart from sessionStorage

	on add item:
			extract productId
			if user is authenticated:
					call backend add-item endpoint
					update local cart cache from response
			else:
					add or increment item in guest cart cache

	on update quantity:
			if quantity <= 0:
					remove item
			else:
					call backend update endpoint or update guest cache

	on remove or clear:
			call backend if authenticated, otherwise update guest cache only
END


4.6 Checkout and Order Placement
-------------------------------
Pseudocode:

START
	user opens checkout page
	verify cart has items
	verify profile has at least one shipping address
	get selected address and payment method
	create order using backend order confirm endpoint
	if payment method is COD:
			clear cart
			load orders
			navigate to profile/orders
	else if payment method is QR:
			clear cart immediately
			open QR payment modal
			wait for payment confirmation callback
			if confirmed:
					reload orders and show success state
	else if payment method is Online:
			open Razorpay flow
			on success:
					reload orders and show confirmation
END


4.7 Order History and Cancellation
----------------------------------
Pseudocode:

START
	load order history from backend
	show filters for status
	if user opens an order:
			show order items, amount, status, and timestamps
	if user cancels an eligible order:
			call cancel endpoint with orderId
			refresh order history
END


4.8 Profile and Address Management
----------------------------------
Pseudocode:

START
	load profile data from backend
	load addresses and normalize casing differences
	if user edits profile:
			submit updated fields to backend
			sync session user object
	if user adds address:
			open address modal
			collect street, city, state, postal code, country, label, default flag
			send address payload to backend
			refresh address list
	if user sets default address:
			update default flag on backend and refresh list
END


4.9 Admin Dashboard and Inventory Management
-------------------------------------------
Pseudocode:

START
	verify logged-in user role is Admin
	open admin dashboard
	fetch customers, orders, and products in parallel
	compute overview metrics such as revenue, order count, customer count, product count
	when admin switches tabs:
			if customers tab:
					list customers
			if orders tab:
					list all orders and allow status updates
			if products tab:
					add, edit, or delete products
			if stock analysis tab:
					calculate low stock and category stock summaries
			if sales report tab:
					load report or compute date range metrics
			if QR payments tab:
					review pending payments and confirm or reject them
END


4.10 Semantic Search Engine
---------------------------
Pseudocode:

START
	receive query and topK
	encode query using sentence-transformer model
	build MongoDB vector search pipeline
	apply price filters inside aggregation when needed
	project numeric fields as doubles
	execute pipeline against products collection
	map results to product DTOs with score and stock data
	return ranked list to caller
END

8. Notes on Workflow Ownership
------------------------------
The frontend owns view state, page navigation, local session restoration, and UI interactions.
The backend owns authentication, persistence, business rules, and the AI bridge routes under /api/ai.
The Python AI agent owns natural language interpretation, semantic product search orchestration, tool selection, and voice transcription.
MongoDB stores users, products, carts, addresses, and orders, while the vector index supports semantic retrieval.
